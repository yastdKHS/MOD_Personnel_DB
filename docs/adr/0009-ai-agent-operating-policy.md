# 0009. AIコーディングエージェント運用方針

## ステータス
Accepted

## コンテキスト

本プロジェクトは長期運用を前提としており、開発の一部（あるいは大部分）がAIコーディングエージェント（Claude Code等）によって行われることが想定される。10年という時間軸では、人間の担当者以上にAIエージェントのセッションは頻繁に入れ替わる。エージェントが毎回文脈をゼロから理解しても、一貫した設計判断ができるようにする仕組みが必要。

## 決定

- リポジトリ直下に、AIエージェント向けの運用規約を明文化した `AGENTS.md`（ツール非依存の汎用規約）と `CLAUDE.md`（Claude Code固有の補足）を設置する。
- 設計判断が必要な変更（データモデル・レイアウト定義フォーマット・ドメイン知識のスキーマ・パイプライン構成・依存関係）は、エージェントが独断で決定せず、既存ADRの確認、または新規ADR提案を経由させる。
- 破壊的操作（force push、履歴書き換え、大量データの一括自動書き換え、pre-commit/CIのバイパス）は、明示的な人間の指示がない限り禁止する。
- 個人情報・データ倫理に関わる判断（[ADR-0008](0008-data-ethics-policy.md)）の範囲を広げる変更は、エージェントが独断で行わない。

## 検討した代替案

- **エージェント向けの規約を作らず、都度の指示のみに頼る**: セッションが変わるたびに規約の再説明が必要になり、一貫性が失われるリスクが高い。長期運用にはスケールしないため見送り。

## 結果（トレードオフ）

- 規約ファイルのメンテナンス自体が継続的なコストになる。規約が実態と乖離しないよう、運用ルールを変更した際は `AGENTS.md` / `CLAUDE.md` も同じPRで更新する。
- ガードレールを設けることで、AIエージェントによる開発速度は一部制約されるが、10年運用における一貫性・安全性を優先する。
