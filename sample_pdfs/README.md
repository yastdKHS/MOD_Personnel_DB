# sample_pdfs/

## 責務

テスト（特に `tests/golden`）で使う、代表的なサンプルPDFを保持する場所。パイプラインが「実際のPDFを最後まで正しく処理できるか」を検証するための入力データ。

## 収録基準

- 防衛省が**一般に公表した**人事発令PDFのうち、レイアウト（様式）の違いを代表する最小限のサンプルのみを収録する。
- 網羅的な全件収録は行わない（リポジトリの肥大化を避けるため）。大量の実データは本リポジトリではなく、別途データストア（`data/` 等、`.gitignore` 対象）で扱う想定。
- 収録するPDFは、`layouts/` の各 `<era_id>` に対して最低1件、対応関係が分かる形で用意する。

## 命名規則

```
<era_id>_<公表日:YYYYMMDD>_<簡単な説明>.pdf
```

例: `2022_format_b_20220401_sample.pdf`

`era_id` は `layouts/` のディレクトリ名と一致させる。

## 出典・ライセンスに関する注意

- ここに収録するPDFは、防衛省が公務として一般公表した情報に限る（[ADR-0008](../docs/adr/0008-data-ethics-policy.md)）。
- 追加時は、取得元URL・公表日をPR説明に明記する。
- 個人情報の取り扱い方針は [ADR-0008](../docs/adr/0008-data-ethics-policy.md) に従う。目的はレイアウト検証であり、収録件数を必要以上に増やさない。

## 対応する期待出力

各サンプルPDFに対応する期待出力（ゴールデンファイル）は `sample_outputs/` に、同じファイル名（拡張子違い）で格納する。
