# scripts/

## 責務

`src/` のパッケージAPIとしては提供しない、定型化されていない運用・保守・開発補助スクリプトを置く場所。

## `src/` との違い

| | `src/` | `scripts/` |
|---|---|---|
| 位置づけ | 安定したパブリックAPI・CLI | 一回限り・不定期の運用作業 |
| テスト | 必須 | 任意（重要なものは推奨） |
| 型チェック対象 | 必須 | 任意 |
| 後方互換性 | 維持する | 維持しなくてよい |

## 想定するスクリプト例（未実装）

- `fetch_sample_pdfs.py`: 公開情報源から `sample_pdfs/` 用の代表サンプルを収集する補助
- `regenerate_golden_outputs.py`: `sample_outputs/` のゴールデンファイルを、レビュー前提で再生成する
- `backfill.py`: 過去分PDFの一括再処理（レイアウト定義追加後の再パース等）

## 方針

- `src/` のコードを呼び出す形で実装し、ロジックの重複を作らない（スクリプトは薄いオーケストレーション層に留める）。
- 破壊的な操作（DBの一括書き換え等）を行うスクリプトは、実行前に対象件数・影響範囲を表示し、確認なしに実行されないようにする。
