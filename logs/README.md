# logs/

## 責務

パイプライン実行時のログ出力先。デバッグ・監査・障害対応のための実行記録を残す場所。

## 方針

- 本ディレクトリの中身（`README.md` と `.gitkeep` を除く）は `.gitignore` によりGit管理対象外とする。ログはリポジトリではなく、実行環境側で保持・ローテーションする。
- ログには実行日時・対象PDF（ハッシュ等の識別情報）・処理件数・検証NG件数などを構造化形式（例: JSON Lines）で出力する想定。個人情報を含むログの取り扱いは [ADR-0008](../docs/adr/0008-data-ethics-policy.md) の方針に従う。
- 具体的なログフォーマット・出力先設定は実装着手時に確定する。

## 保持しているファイル

- `.gitkeep`: 空ディレクトリをGit管理下に置くためのプレースホルダ。
