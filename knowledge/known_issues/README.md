# knowledge/known_issues/

## 責務

PDF記載内容の既知の癖・例外パターン（誤記、表記の不整合、特殊な発令形式等）と、それらへの対応方針を記録する。

## 想定する情報

- 発見された例外パターンの説明と、確認できる出典（該当PDF・日付）
- 現状の対応方針（自動対応済み／人手確認が必要／未対応）
- 関連する `layouts/` や `knowledge/` 内の他データへの参照

## 方針

- ここは「バグトラッカー」ではなく、ドメイン特有の恒常的な知識の蓄積場所である。個別の実装タスクの進捗管理はGitHub Issuesで行い、ここには再発しうる一般的な知見を残す。
- 具体的なファイル形式は実装着手時に確定する。
