# sample_outputs/

## 責務

`sample_pdfs/` の各サンプルPDFに対応する、パイプラインの期待出力（ゴールデンファイル）を保持する場所。設計判断の背景は [ADR-0007](../docs/adr/0007-golden-file-testing.md) を参照。

## 命名規則

`sample_pdfs/` のファイル名から拡張子のみを変えたものとする。

```
sample_pdfs/2022_format_b_20220401_sample.pdf
  → sample_outputs/2022_format_b_20220401_sample.json
```

## 内容（想定）

パイプラインの `parse` → `normalize` を経た後の構造化データ（人物・階級・補職・組織・発令日等）を、人間が読める形式（JSON等、確定は実装時）で保持する。

## 更新ルール

- ゴールデンファイルの更新は「テストを通すための機械的な作業」ではなく「何が正しい抽出結果かを再定義する行為」である。
- 更新する場合は、なぜ期待値が変わったのか（レイアウト定義の修正、正規化ロジックの改善等）をPR説明に明記する。
- 無言でのゴールデンファイル書き換えは `CLAUDE.md` / `AGENTS.md` で禁止事項として扱う。
