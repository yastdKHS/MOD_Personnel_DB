# tmp/

## 責務

パイプライン実行中の一時的な中間生成物（ダウンロード直後のPDF、検証前の中間JSON等）を置くスクラッチ領域。

## 方針

- 本ディレクトリの中身（`README.md` と `.gitkeep` を除く）は `.gitignore` によりGit管理対象外とする。
- ここに置かれるものは常に再生成可能でなければならない。恒久的に必要なデータ（サンプル・成果物）を `tmp/` に置いてはならない。
- 実行環境によっては定期的に自動削除される想定。長期保持が必要なものは `sample_pdfs/` / `sample_outputs/` / 正データストアに移す。

## 保持しているファイル

- `.gitkeep`: 空ディレクトリをGit管理下に置くためのプレースホルダ。
